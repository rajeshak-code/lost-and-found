About the Project

Lost & Found Portal – A simple yet effective solution for lost & found items in campuses and communities.

Project Objective

The main goal of this web application is to provide a centralized platform where users can:

  • Report items they have lost
  • Report items they have found
  • Browse listings of lost and found items
  • Contact the owner/finder directly to recover/return belongings

This helps reduce time wasted searching physically and increases the chances of reuniting people with their lost items — especially useful in colleges, universities, hostels, buses, events, and public places.

Technologies Used

Frontend
HTML5, CSS3 (Grid + Flexbox), JavaScript (Vanilla)
Styling
Pure CSS (no frameworks like Bootstrap)
Data Storage (Demo)
JavaScript Array + localStorage simulation
Hosting
GitHub Pages (Static site)

Future enhancements (planned for full-stack version): Node.js/Express backend, MongoDB or Firebase database, user authentication, email notifications, image upload to cloud storage.

Advantages

  • Centralized reporting → no need to post in multiple WhatsApp groups or notice boards
  • Quick search and filter by location/category/date
  • Direct contact between finder and owner
  • Reduces environmental waste (fewer replacement purchases)
  • Community helpfulness — people feel good returning items
